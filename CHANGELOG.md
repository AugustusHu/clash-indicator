# Changelog

## 1.4.1 (2026-07-23)

- 向导页接入中文、English 和跟随浏览器三种语言模式
- 补充路由器、本机 external controller 和 Zashboard `/ui/` 地址填写说明
- 向导页新增 BLOCK 图例、兼容核心说明和深色模式
- 更新后首次打开 Popup 显示一次版本提示，设置区域常驻显示当前版本号
- 用户可见文案统一强调 Clash-compatible API，明确支持 Clash、Mihomo 和 sing-box
- 新增可从 Popup 随时打开的使用帮助页，提供 Clash、Mihomo、sing-box 配置示例、地址定位方法和常见问题
- 新增繁体中文和日语，并完善跟随浏览器时的语言识别
- 面板地址支持独立配置，兼容三种核心的内置面板及单独部署的 Zashboard

## 1.4.0 (2026-07-12)

### 新特性

- 页面资源去向升级为三列表格，展示域名、连接方式、平均请求耗时和请求次数
- 新增 BLOCK 分类，识别 Clash/Mihomo 的 REJECT、REJECT-DROP、DROP、BLOCK，以及浏览器明确报告的拦截错误
- 新增请求期间路由采样，减少短连接结束后被归为“未知”的情况
- 新增中文、English 和跟随浏览器三种界面语言选项，切换后立即生效并同步保存
- 弹窗支持自动深浅色主题和轻微半透明效果

### 修复与调整

- 修复小写 `direct` 被误判为代理的问题
- 修复使用同一主域名下其他连接推断当前页面时可能产生的路由误判
- 修复自动语言模式读取旧语言资源、显示 `languageLabel` 等内部 key 的问题
- 主页面命中拦截规则时显示 BLOCK，不再显示为代理
- 移除页面上下行流量展示及对应的后台聚合计算

## 1.3.0 (2026-06-12)

- 移除弹窗内切换节点功能（保持纯只读，降低权限敏感度）
- 新安装自动打开引导页：固定扩展到工具栏 + 配置 controller 地址和 secret
- 未配置时弹窗和图标显示中性"待配置"提示并自动展开设置，不再显示红色连接失败

## 1.2.0 (2026-06-12)

- 中英双语（chrome.i18n，跟随浏览器语言）
- controller 地址支持 https，未写协议自动补全
- 默认 controller 地址改为 http://192.168.10.1:9090
- 更名为「Clash 指示器 / Clash Indicator」

## 1.1.0 (2026-06-12)

- 图标动态变色（绿直连 / 蓝代理 / 灰未知 / 红错误）
- 弹窗显示节点延迟、页面流量
- 页面资源去向汇总（webRequest 域名追踪 + 比例条）
- 弹窗内切换代理组节点
- 打开 Clash 面板入口

## 1.0.0 (2026-06-12)

- 首个版本：角标显示直连/代理，弹窗显示命中规则和节点链，controller 地址与 secret 可配置
